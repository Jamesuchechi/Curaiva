MCP Superpower — Option 1

Six clinical tools.
Any agent can call them.

Each tool fetches real patient data from a FHIR R4 server via SHARP context, processes it through Claude AI, and returns structured clinical JSON.

Tool 01
triage_patient()
Assess symptoms against FHIR patient history. Returns severity score (1–10), likely conditions, self-care steps, and escalation flag.
Patient Condition Observation
Tool 02
get_patient_summary()
Fetches all FHIR resources for a patient and generates a brief, full, or pre-consultation clinical summary via Claude.
Patient Condition MedRequest Encounter
Tool 03
check_medication_adherence()
Analyses active MedicationRequest resources. Scores adherence risk 0–100. Recommends CHW alert when risk is high.
Patient MedicationRequest
Tool 04
mental_health_assessment()
Detects mood patterns and crisis indicators from session notes. Returns crisis_flag: true with emergency escalation if needed.
Patient Observation (survey)
Tool 05
generate_chw_priority_queue()
Fetches up to 20 patients in parallel, scores each by urgency, and returns a ranked watchlist for Community Health Workers.
Patient Condition MedRequest Observation
Tool 06
create_consultation_brief()
Builds a physician-ready pre-consultation document from FHIR data. Includes chief complaint, active problems, meds, vitals, and focus areas.
Patient Condition MedRequest Observation

🔌

Zero custom token handling

Prompt Opinion bridges EHR session credentials into SHARP context automatically. Our tools just read the values and use them.

📡

Works with any FHIR R4 server

Tested against HAPI FHIR public server. Deployable against any compliant EHR — Epic, Cerner, Smile CDR — via the fhir_base_url in SHARP context.

⚡

Parallel FHIR fetching

Tools like generate_chw_priority_queue fetch up to 20 patients simultaneously using Promise.allSettled — results in under 3 seconds.

🔐

No data stored

Curaiva AI fetches FHIR data transiently at query time. Nothing is persisted. Deploying organizations own their data entirely.
